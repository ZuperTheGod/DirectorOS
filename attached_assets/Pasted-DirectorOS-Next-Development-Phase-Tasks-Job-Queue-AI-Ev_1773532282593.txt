DirectorOS Next Development Phase
Tasks: Job Queue + AI Evaluation + Clip Library

Goal: upgrade DirectorOS into a reliable AI video production pipeline capable of batch generation, quality scoring, and reusable assets.

TASK 1 — Generation Job Queue System
Purpose

Currently generation tasks run synchronously through API routes. This can block the server and does not scale.

Implement an asynchronous job queue so that image/video generation runs as background workers.

Architecture
User Action
↓
API Route
↓
Create Generation Job
↓
Queue Worker
↓
Run AI Connector
↓
Update Job Status
↓
Notify UI
Database Schema

Create table:

generation_jobs

Fields:

id
project_id
shot_id
type (image | video | render)
status (pending | processing | completed | failed)
input_data JSON
output_asset_id
error_message
created_at
updated_at
Backend Modules

Create:

services/job-queue/

Files:

queue.ts
worker.ts
job-types.ts
queue.ts

Responsible for adding jobs.

Example:

export async function enqueueJob(jobType, payload) {
  return db.insert(generationJobs).values({
    type: jobType,
    status: "pending",
    inputData: payload
  });
}
worker.ts

Runs background job processing loop.

Responsibilities:

poll pending jobs
mark job processing
run connector
store result
update job status

Pseudo:

while(true) {
  const job = await getNextPendingJob();

  if(job) {
    processJob(job);
  }

  sleep(2000);
}
Supported Job Types
IMAGE_GENERATION
VIDEO_GENERATION
TIMELINE_RENDER
API Changes

Instead of directly generating assets, these routes should now enqueue jobs:

POST /shots/:id/generate-image
POST /shots/:id/generate-video
POST /projects/:id/export

Response:

{
  "jobId": "...",
  "status": "queued"
}
